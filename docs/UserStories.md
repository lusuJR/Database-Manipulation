# SchoolDB User Stories

## Week 1

### User Story

As a Database Administrator,

I want to create the SchoolDB database

so that student information can be stored securely.

---

## Week 2

### User Story

As a Principal,

I want to generate student and course reports

so that I can monitor student registrations.

---

## Week 3

### User Story

As a Registrar,

I want to safely insert, update and delete student records

so that the database remains accurate and reliable.

---

## Week 4

### Views

As a Principal,

I want to view student enrolment reports

so that I can analyse registrations without writing complex SQL queries.

---

### Stored Procedures

As a Registrar,

I want to search for a student using a student number

so that I can quickly retrieve student information.

---

### User-Defined Functions

As a Principal,

I want reusable calculations

so that reports remain consistent throughout the system.

---

### Triggers

As a Database Administrator,

I want business rules to be enforced automatically

so that invalid database operations are prevented.
